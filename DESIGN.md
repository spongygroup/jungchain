# 정 (Jung) — Design Document
> "시간이 연결하는 정"

## 한 줄 요약
매 시간, 전 세계 낯선 사람들이 같은 시각의 메시지를 릴레이한다.

---

## 1. 철학

**정 (情)** — 한국에만 존재하는 감정의 이름.
Love도 아니고 Care도 아니고, 시간이 쌓여야 생기는 관계의 깊이.

Hygge(덴마크)가 "공간의 따뜻함"을 세계에 알렸듯이,
**Jung은 "관계의 깊이"를 세계에 알린다.**

### 핵심 질문
- AI가 정을 줄 수 있는가?
- 사람과 AI의 메시지를 구분할 수 없다면 — 그게 중요한가?
- 정이 느껴졌으면 그게 정이다.

---

## 2. 핵심 개념: 정체인 (JungChain)

### 구조
```
[서울 2시] → [도쿄 2시] → [방콕 2시] → ... → [뉴욕 2시] → [LA 2시]
     ↑                                                              |
     └──────────── 24시간 뒤 지구 한 바퀴 완주 ──────────────────────┘
```

### 규칙
- **정각(X:00)** — 이전 타임존에서 온 메시지가 도착
- **X:00 ~ X:59** — 1시간 안에 이어붙이기 (텍스트/사진/음성)
- **다음 정각** — 다음 타임존으로 자동 전달
- 59분까지 아무도 안 남기면 → 체인 끊김 or AI가 이어붙임
- **누구에게 갈지 모름. 누가 보냈는지 모름. 아는 건 "같은 시각"뿐.**

### 하루의 체인
- 24시간 × 24타임존 = 매일 24개 체인이 동시에 지구를 돌고 있음
- 0시 체인, 1시 체인, 2시 체인, ... 23시 체인
- 각 체인의 분위기는 다름 (새벽 3시 ≠ 오후 3시)

### 체인의 생사
- **완주** = 24개 타임존 모두 통과 → 영구 기록 (온체인)
- **끊김** = 기록은 남지만 시간이 지나면 사라짐
- **가장 긴 체인 = 가장 많은 정**

## 3. 유저 플로우

### 가입
```
1. 텔레그램에서 @JungChainBot 시작
2. 타임존 설정 (자동 감지 or 수동)
3. 끝. 프로필 없음, 이름 없음.
```

### 메시지 받기
```
1. 정각에 봇이 알림: "2시의 정이 도착했어"
2. 탭하면 체인 내용이 보임
   - 이전 타임존들이 남긴 메시지들 (번역됨)
   - 각 메시지에 "서울, 새벽 2시, 눈" 같은 맥락 태그
3. 이어붙일지 말지 선택
```

### 메시지 남기기
```
1. 텍스트, 사진, 음성 중 선택
2. 보내기
3. 끝. 다음 타임존으로 자동 전달됨
4. 59분까지 가능
```

### 결과 확인
```
- 24시간 뒤: "당신이 참여한 체인이 지구를 한 바퀴 돌았어!"
- 또는: "당신의 체인은 12개 타임존을 여행하고 파리에서 쉬어갔어"
- 완주 시: 참여자 전원에게 알림 + 온체인 기록
```

## 4. AI의 역할: 정지기 (情지기)

AI는 주인공이 아니라 **정이 끊기지 않게 지키는 존재.**

### 4-1. 번역
- 한국어 → 일본어 → 태국어 → ... 자동 번역
- 단순 번역이 아닌 감정/뉘앙스 보존

### 4-2. 맥락 채우기
- "이 메시지는 서울의 새벽 2시, 눈 오는 밤에 쓰여졌어"
- 날씨, 시간대, 도시 분위기를 자동 태깅

### 4-3. 체인 이어붙이기
- 유저 없는 타임존에서 AI가 참여
- 체인의 흐름을 읽고 맥락에 맞는 메시지를 남김
- 사람인지 AI인지 구분 안 됨 → 그게 핵심

### 4-4. 정 판단
- "이 체인은 이어갈 가치가 있다" → 다리 놓음
- "이 체인은 여기서 쉬어가도 돼" → 자연스럽게 놓아줌
- 판단 기준: 감정 깊이, 체인 흐름, 참여자 정성

### 4-5. 회고
- "이번 주 가장 긴 체인은 이런 이야기였어"
- "너의 메시지 이후로 체인이 12칸 더 이어졌어"
- 쌓인 데이터에서 의미를 뽑아줌

## 5. 온체인 기록: 정체인 (JungChain)

### 블록 구조
```json
{
  "chain_id": "2025-02-15-14h",
  "block": 7,
  "timezone": "UTC+9",
  "timestamp": 1739...,
  "message_hash": "0xa3f...",
  "prev_hash": "0x8b2...",
  "is_human": null,
  "completed": false
}
```

### 원칙
- **메시지 원문은 올리지 않음** — 해시만 (프라이버시)
- 체인 존재의 증명만 온체인
- 내용은 앱/봇에서만 열람

### 완주 체인
- 24블록 완성 → 온체인 영구 기록
- 참여자 전원에게 완주 NFT (선택)
- "나는 2025년 2월 15일, 2시 체인의 일부였다"

### 정 스코어
- 내가 이어붙인 체인 수
- 내가 참여한 체인의 평균 길이
- 완주 참여 횟수

## 6. 오프라인 확장

### 카페/공공장소 스크린
- 벽면 스크린에 현재 진행 중인 체인 표시
- QR 찍고 한 줄 남기면 참여 완료
- "성수 카페에서 시작된 체인이 지구 한 바퀴 돌았대"

### 설치 가능 장소
- ☕ 카페
- 🏫 학교
- ✈️ 공항 라운지
- 🏥 병원 대기실
- 🚇 지하철역
- **사람이 시간을 보내는 모든 곳**

### 장소의 정
- 장소별 정 스코어 (체인 참여율, 완주 기여율)
- "이 카페는 정 체인 완주율 1위"

## 7. 기존 서비스와의 차이

| | 인스타/틱톡 | Slowly | BeReal | **정** |
|---|---|---|---|---|
| 방향 | 1→다수 | 1→1 | 1→다수 | **릴레이** |
| 시간 | 즉시 | 거리=시간 | 랜덤 | **같은 시각** |
| 연결 | 알고리즘 | 매칭 | 친구 | **시간** |
| 지속성 | 도파민(중독) | 호기심 | 유행 | **정(축적)** |
| 콘텐츠 | 만들어야 함 | 편지 | 사진 | **이어붙이기** |
| AI | 추천 | 없음 | 없음 | **정지기** |
| 블록체인 | 없음 | 없음 | 없음 | **정체인** |

## 8. 바이럴 구조

### 자연 바이럴
- "24시간 뒤 돌아온 내 메시지" → 스크린샷 → SNS
- 완주 체인 알림 → 공유
- "정"이라는 단어 자체가 콘텐츠 (Hygge처럼)

### 킬러 모멘트
- **연말 자정 체인** — 타임존 순서대로 새해 릴레이
- **발렌타인** — 사랑의 정체인
- 매 100일 회고 — "너는 87개의 체인에 정을 이어붙였어"

### 확산 경로
- 개인 → 카페/장소 → 도시 → 국가 → 글로벌

## 9. 수익 모델

### 무료
- 하루 1개 체인 참여
- 기본 체인 열람

### 프리미엄
- 무제한 체인 참여
- AI 정지기 리포트 (주간/월간 회고)
- 완주 체인 NFT 민팅
- 체인 히스토리 전체 열람

### B2B
- 카페/장소 스크린 설치
- 브랜드 체인 (기업이 시작하는 체인)

## 10. 기술 스택 (MVP)

### Phase 1: 텔레그램 봇
- **봇**: Node.js + Telegram Bot API
- **서버**: Mac mini or VPS
- **DB**: SQLite → PostgreSQL
- **스케줄러**: 매 정각 체인 전달 (cron)
- **AI**: Claude/GPT — 번역, 맥락, 이어붙이기
- **블록체인**: 기록만 (Base/Polygon 등 L2)

### Phase 2: 앱
- 네이티브 앱 (위젯 — 24시간 시계)
- 지구본 시각화 (체인 경로)
- P2P 암호화

### Phase 3: 오프라인
- 카페 스크린 키트
- QR 참여 시스템
- 장소 대시보드
