# 포토 릴레이 수정사항
> 2026-02-15 JB 피드백
> 풀 24블록 완주: 792초 (~13.2분), 에러 0

## 필수 수정

### 1. 다음 턴 유저 경험
- 다음 턴 유저는 **미션 + 직전 유저의 사진&캡션**만 받음
- 받자마자 본인도 사진 찍어서 참여
- 전체 사진 X, 직전 1개만

### 2. 원래 유저(JB)한테 오는 진행 알림
- **텍스트만, 사진 없음**
- 매 블록 알림 X → 간격 알림만:
  - ✈️ 네 사진이 다음 도시로 떠났어.
  - 🌍 네 사진이 [직후 도시]에 도착했어.
  - 🌍 사진이 [도시]를 지나는 중... (5/24)
  - 🌍 사진이 [도시]를 지나는 중... (10/24)
  - 🌍 사진이 [도시]를 지나는 중... (15/24)
  - 🌍 사진이 [도시]를 지나는 중... (20/24)
  - 🌍 거의 다 왔어! [마지막 도시]까지.
- 5블록 간격 + 직후 도시 + 마지막만

### 3. 최종 결과 — 사진 갤러리
- 현재: 텍스트(도시+캡션) 목록만 전송 → 사진 없음
- 목표: 완주 후 **24장 사진 모아서 보여주기**
  - 방법 후보: Telegram 미디어 그룹 (최대 10장씩), 또는 웹 갤러리 링크
  - 각 사진에 도시명+캡션 포함

### 4. 이미지 로컬 저장
- 현재: imageBuffer가 메모리에만 있다가 프로세스 종료 시 소실
- 목표: 매 블록 생성 즉시 `data/relay-photos/{chainId}/{block}.jpg`로 저장
- 캡션/도시/메타데이터도 JSON으로 함께 저장
- 최종 콘텐츠 생성에 필수

### 5. 사진 스타일 — 일상감 필요
- 현재: Imagen 4가 스톡포토급 고퀄 → "정" 느낌 안 남
- 목표: **실제 사람이 폰으로 찍은 듯한 평범한 일상 사진**
  - JB의 타바스코 사진처럼 자연스러운 느낌
  - 완벽한 구도/조명 X → 약간 대충, 생활감 있게
  - 비율도 랜덤 (정사각, 세로, 가로 섞여도 OK)
- 방법: 이미지 프롬프트에 "casual phone photo", "slightly messy", "everyday life" 등 추가
  - "professional photography" 류 표현 제거
