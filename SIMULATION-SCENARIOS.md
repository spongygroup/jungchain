## 2. 체인 끊김 시나리오

### 2-A. 유저 없는 타임존

**상황:** UTC+6 (방글라데시) — 등록 유저 0명

```
[블록 N] UTC+7 방콕 02:45
  🧑 "วันนี้เหนื่อยมาก แต่ยิ้มได้เพราะเพื่อนๆ ในเชนนี้"

[블록 N+1] UTC+6 다카 — 유저 없음
  ⏰ 02:00 체인 도착
  ⏰ 02:15 매칭 시도 → 유저 없음
  🤖 AI 정지기 판단: 감정 흐름 따뜻함(0.8) + 체인 길이 11 → 이어붙이기
  🤖 "피곤한 하루 끝에 웃을 수 있는 건, 누군가 곁에 있다는 느낌 때문이야."
  → 번역(영): "Being able to smile at the end of a tiring day — it's because someone feels close."

[블록 N+2] UTC+5 카라치 02:22
  🧑 (정상 유저 참여)
```

### 2-B. 유저 있지만 59분까지 미응답

```
[블록 N] UTC+1 파리 02:00
  📩 유저 @user_fr에게 알림: "2시의 정이 도착했어"
  ⏰ 02:00 알림 발송
  ⏰ 02:30 리마인더: "아직 시간 있어. 한 마디면 돼."
  ⏰ 02:55 최종 리마인더: "5분 남았어."
  ⏰ 02:59 타임아웃

  → AI 정지기 판단:
    - 체인 감정 깊이: 높음 (위로 흐름)
    - 체인 길이: 15/24
    - 판단: 이어붙이기
    🤖 "파리의 새벽은 조용하네. 누군가 잠들어 있을 거야. 대신 이 고요함을 전할게."
```

### 2-C. AI가 이어붙이는 vs 끊기는 판단

**이어붙이는 케이스:**
```
조건:
  - 체인 길이 ≥ 12 (절반 이상)
  - 감정 흐름이 연속적 (위로→감사→응원)
  - 남은 타임존 ≤ 5

AI: "여기서 끊기엔 너무 아까워. 조금만 더 가보자."
→ 이어붙임
```

**끊기는 (놓아주는) 케이스:**
```
조건:
  - 체인 길이 = 3 (초반)
  - 메시지 내용이 단답 ("ㅋ", "ok", ".")
  - 감정 깊이 낮음

AI 판단: "이 체인은 여기서 쉬어가도 괜찮아."
→ 체인 종료

참여자 알림:
  "당신의 체인은 3개 타임존을 여행하고 카이로에서 쉬어갔어.
   내일 다시 만나자. 🌙"
```

```

02:08 @user_tokyo1: "東京も寒いよ。みんな起きてるんだね。"

02:15 @user_osaka1: "大阪から。チェーンに参加できて嬉しい。"
02:22 @user_seoul3: (사진: 창밖 눈 내리는 서울 야경)
```

### 처리 방식

**옵션 A: 먼저 온 1개만 전달 (MVP)**
```
규칙: 가장 먼저 도착한 메시지 = 체인의 공식 블록
나머지: "가지(branch)" 메시지로 기록 — 체인 열람 시 함께 표시

→ 02:05 @user_seoul1 메시지가 블록으로 선택
→ 나머지 4개는 가지 메시지 (체인 뷰에서 "이 시간대 +4개" 표시)
```

**옵션 B: 투표/랜덤 (향후)**
```
- 59분까지 모든 메시지 수집
- 가장 "정이 깊은" 메시지를 AI가 선택
- 또는 랜덤
```

### 전달 시 표시
```
다음 타임존이 받는 화면:

━━━ 서울/도쿄, 새벽 2시, 눈 ━━━
"눈 오는 밤이네. 이 체인 따뜻하다."

  💬 이 시간대에 4명이 더 참여했어
  [펼쳐보기]
```

## 4. AI 정지기 시나리오

### 4-A. 이어붙이는 판단

```
체인 맥락:
  블록 10: "今日、おばあちゃんが亡くなった。" (오늘 할머니가 돌아가셨다)
  블록 11: "I'm so sorry. Sending you warmth from London."
  블록 12: "حزين جدًا. الله يرحمها." (너무 슬프다. 편히 쉬시길.)
  블록 13: (유저 없음)

AI 판단:
  - 감정: 애도/위로 (깊음)
  - 흐름: 연속적, 진심 있음
  - 결정: 반드시 이어붙인다

AI 메시지:
  🤖 "말이 없어도 괜찮아. 지금 이 조용함도 함께 있다는 뜻이야."
  → 英: "Silence is okay. This quietness is also a way of being together."
  → 日: "言葉がなくても大丈夫。この静けさも、一緒にいるということだから。"
```

### 4-B. 놓아주는 판단

```
체인 맥락:
  블록 1: "아 졸려"
  블록 2: "same"
  블록 3: "😴"
  블록 4: (유저 없음)

AI 판단:
  - 감정: 중립/낮음
  - 흐름: 단답, 에너지 낮음
  - 체인 초반 (4/24)
  - 결정: 놓아준다

→ 체인 종료
→ "이 체인은 4개 타임존을 조용히 여행하고 쉬어갔어."
```

### 4-C. 맥락 채우기 예시

```
유저 메시지: "비 온다"

AI가 채우는 맥락 태그:
  🏷️ 서울, 새벽 2시
  🌧️ 실제 날씨: 비 (기상청 API)
  🌡️ 2°C
  📍 UTC+9

다음 타임존에 전달 시:
  ━━━ 서울, 새벽 2시, 비 2°C ━━━
  "비 온다"
  → 英: "It's raining."
  → 맥락이 감정을 만든다: 새벽 2시 + 비 + 2도 = 쓸쓸함
```

### 4-D. AI 메시지 퀄리티 (감정별)

**따뜻함** (체인이 응원/감사 흐름일 때)
```
🤖 "여기까지 온 메시지들, 하나하나 따뜻하다. 다음 사람에게도 이 온기가 닿길."
→ "Every message that's made it this far carries warmth. May it reach the next person too."
```

**위로** (슬픔/힘듦 흐름)
```
🤖 "아무 말 안 해도 돼. 그냥 여기 있을게."
→ "You don't have to say anything. I'm just here."
→ "何も言わなくていいよ。ただここにいるから。"
```

**유머** (가벼운/장난 흐름)
```
🤖 "이 체인 보면서 웃고 있는 사람이 지금 최소 3개 대륙에 있을 듯 ㅋㅋ"
→ "At least three continents are probably laughing at this chain right now 😂"
```

**조용함** (새벽/명상적 흐름)
```
🤖 "..."
→ (점 세 개만. 때로는 침묵이 가장 깊은 메시지.)
```

## 5. 엣지 케이스

### 5-A. 부적절한 메시지

```
블록 N: "f*** you all this is stupid"

AI 정지기 판단:
  - 감정 분석: 적대적
  - 체인 맥락과 불일치
  - 처리:
    1. 해당 메시지 필터링 (다음 타임존에 전달 안 함)
    2. 유저에게: "이 메시지는 체인의 흐름과 맞지 않아서 전달되지 않았어.
                  다시 써볼래?"
    3. 체인은 유저 없음 처리 → AI 이어붙이기 판단으로

스팸/광고:
  블록 N: "Buy crypto at www.scam.com!!!"
  → 자동 필터 → 전달 차단 → 체인 흐름 유지
```

### 5-B. 0시 전체 사이클

```
체인 ID: 2025-02-15-00h (자정 체인)

Day 1 — 02/15
  00:00 UTC+12 (뉴질랜드) → 체인 시작
  01:00 UTC+11
  02:00 UTC+10 (호주)
  ...
  12:00 UTC+0 (런던) → 절반 지점
  ...

Day 2 — 02/16
  ... (UTC-10, -11 등 날짜 넘어감)
  23:00 UTC-11 (사모아) → 완주

총 소요: ~24시간
참여자들의 체감: "어제 자정에 시작한 체인이 오늘 돌아왔다"
```

### 5-C. 같은 유저의 다중 참여

```
@user_nomad: 디지털 노마드, 실제 위치 = 태국 (UTC+7)
등록 타임존: UTC+7

상황: 여행 중 일본(UTC+9)에 있음

규칙:
  - 등록 타임존 기준으로 참여 (UTC+7)
  - 같은 유저가 하루에 여러 체인 참여 가능? → MVP에서는 1체인/일
  - 프리미엄: 무제한

엣지:
  @user_nomad가 0시 체인(UTC+7)과 3시 체인(UTC+7) 둘 다 참여?
  → 무료: 첫 번째만 가능
  → 프리미엄: 둘 다 가능, 단 같은 체인에 2번은 불가
```

### 5-D. 30분 오프셋 타임존

```
UTC+5:30 (인도) / UTC+5:45 (네팔) / UTC+9:30 (호주 중부)

처리 방식 (MVP):
  - 가장 가까운 정시 타임존에 병합
  - UTC+5:30 → UTC+5 또는 UTC+6 그룹에 합류
  - 유저에게: "가장 가까운 시간대(UTC+6)에 배정됐어"

처리 방식 (향후):
  - 30분 오프셋 전용 슬롯 추가
  - UTC+5 → UTC+5:30 → UTC+6 순서
  - 체인이 48스텝으로 확장 (30분 단위)

인도 유저 시뮬레이션:
  등록: 타임존 = UTC+5:30
  시스템: "UTC+5:30은 UTC+6 그룹에 배정됩니다"
  유저: 02:00(UTC+6) = 01:30(인도 시간)에 알림 받음
  → 주의: "새벽 1시 30분에 알림? 괜찮을까?"
  → 해결: 유저가 선호 시간 선택 가능 (2시 or 3시 체인)
```
